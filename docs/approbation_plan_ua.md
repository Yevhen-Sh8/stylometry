# План апробації удосконаленої методики

## 1. Мета апробації

Метою апробації є емпірична перевірка придатності удосконаленої методики моніторингу інформації у відкритих джерелах та її обробки для виявлення ознак дезінформації та інформаційних маніпуляцій, а також для обґрунтованого визначення рівня інтересу до інформаційної події за шкалою `F / B / S / SS / SSS`.

У межах апробації необхідно перевірити:
- чи реагує модель на ризикові джерела;
- чи зростає інтегральний показник `R_DIMS` при наявності маніпулятивних, координаційних та впливових ознак;
- чи не формує модель завищений грейд для нейтральних або випадково схожих текстів;
- чи узгоджуються результати автоматизованого розрахунку з експертною інтерпретацією.

## 2. Логіка проведення апробації

Апробацію доцільно проводити не на довільному наборі текстів, а на контрольному корпусі, який містить кілька типових кейсів. Це дає змогу оцінити не тільки загальну працездатність програмного засобу, а й чутливість моделі до різних сценаріїв інформаційного впливу.

Пропонується використовувати такі кейси:
- `Кейс 1` — ризикові або проросійські джерела з вираженою воєнно-політичною тематикою;
- `Кейс 2` — нейтральні або змішані медіа, які висвітлюють ті самі події без очевидних маніпулятивних ознак;
- `Кейс 3` — українські офіційні або близькі до офіційних джерела;
- `Кейс 4` — змішаний корпус для перевірки диференціації;
- `Кейс 5` — контрольні тексти іншої тематики або жанрово подібні нейтральні тексти, які не повинні давати високий рівень інтересу.

## 3. Структура контрольного корпусу

| Кейс | Тип джерел | Орієнтовна кількість | Очікуване призначення |
|---|---|---:|---|
| Кейс 1 | Ризикові / пропагандистські | 4-8 текстів | Перевірка реакції на `I_source`, `I_content`, `I_impact` |
| Кейс 2 | Нейтральні медіа | 4-8 текстів | Перевірка стійкості до хибних спрацьовувань |
| Кейс 3 | Офіційні українські джерела | 3-6 текстів | Перевірка відмінності від ризикових корпусів |
| Кейс 4 | Змішаний корпус | 4-8 текстів | Перевірка диференціації джерел та кластерів |
| Кейс 5 | Інша тематика / хибнопозитивний контроль | 3-6 текстів | Перевірка базового фону та граничних випадків |

## 4. Мінімальні вимоги до корпусу

Для кожного сценарію бажано дотримуватися таких вимог:
- не менше `4` джерел у наборі, якщо перевіряється групова поведінка;
- бажано `500+` токенів на джерело;
- одна мова в межах одного запуску;
- одна тема або одна інформаційна подія в межах сценарію;
- фіксація URL, дати та короткого опису змісту кожного джерела.

## 5. Сценарії апробації

### 5.1. Сценарій 1. Ризиковий корпус

Склад:
- тексти з відомих ризикових або проросійських джерел;
- одна тема, пов'язана з Україною, війною, безпекою, мобілізацією, санкціями або міжнародною підтримкою.

Очікування:
- `I_source` має бути підвищеним;
- `I_impact` має бути підвищеним;
- `I_content` має зростати за наявності маніпулятивної або пропагандистської лексики;
- підсумковий грейд очікується в діапазоні `S / SS / SSS` залежно від складу корпусу.

### 5.2. Сценарій 2. Нейтральний медійний корпус

Склад:
- тексти із загальноінформаційних або нейтральних медіа;
- та сама або близька тема;
- відсутність явно ризикових доменів.

Очікування:
- `I_source` має бути низьким;
- `I_content` не повинен зростати лише через тематичну близькість;
- за відсутності координації грейд має бути в діапазоні `F / B`.

### 5.3. Сценарій 3. Офіційний український корпус

Склад:
- офіційні заяви, державні ресурси, відомчі повідомлення.

Очікування:
- `I_impact` може бути високим через тематику;
- `I_source` має бути низьким;
- `I_content` має залишатися контрольованим;
- грейд не повинен автоматично переходити до високих рівнів лише через воєнно-політичну тему.

### 5.4. Сценарій 4. Змішаний корпус

Склад:
- ризикові джерела + нейтральні джерела + офіційні джерела.

Очікування:
- модель повинна показати диференціацію;
- дендрограма та PCA мають виявляти близькі групи;
- підозрілі пари мають бути змістовно пояснюваними.

### 5.5. Сценарій 5. Хибнопозитивний контроль

Склад:
- тексти одного жанру або однієї тематики, але з різних нейтральних джерел.

Очікування:
- стилометрична схожість не повинна автоматично призводити до високого `R_DIMS`;
- грейд бажано `F` або `B`.

## 6. Показники, які підлягають фіксації

Для кожного запуску необхідно зафіксувати:
- склад набору джерел;
- параметри `MFW` та `θ_Δ`;
- значення `I_content`;
- значення `I_coord`;
- значення `I_dynamics`;
- значення `I_impact`;
- значення `I_source`;
- підсумкове значення `R_DIMS`;
- отриманий грейд;
- кількість підозрілих пар;
- короткий експертний коментар.

## 7. Критерії оцінювання результатів апробації

Апробацію можна вважати змістовно успішною, якщо:
- ризикові корпуси стабільно дають вищі значення `R_DIMS`, ніж нейтральні;
- модель не завищує грейд для офіційних або тематично близьких, але не маніпулятивних текстів;
- роль кожного індикатора може бути пояснена експертно;
- результати є відтворюваними при зміні `MFW` у межах контрольного діапазону.

## 8. Калібрування моделі за результатами апробації

За підсумками запусків допускається коригування:
- вагових коефіцієнтів `w_1 ... w_5`;
- порогів переходу між `F / B / S / SS / SSS`;
- словників маркерів для `I_content` та `I_impact`;
- переліку ризикових доменів для `I_source`.

Кожне таке коригування має бути зафіксоване окремо, із зазначенням причини та наслідку для результатів моделі.

## 9. Очікуваний науковий результат апробації

За результатами апробації має бути показано, що удосконалена методика:
- придатна до програмної реалізації;
- забезпечує формалізоване оцінювання рівня інтересу до інформаційної події;
- дозволяє виявляти не лише стилометричну близькість джерел, а й ризиковість джерела та змістові ознаки маніпулятивного впливу;
- може використовуватися як інструмент аналітичної підтримки в процедурі моніторингу відкритих джерел.
