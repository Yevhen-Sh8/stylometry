# Дизайн-система DIMS

**DIMS** (*Disinformation Information Monitoring Score* — «Інтегральна оцінка дезінформаційних ризиків за стилометричними ознаками») — це аналітичний застосунок державного рівня для моніторингу інформації у відкритих джерелах та виявлення скоординованої дезінформації. Він застосовує стилометричну метрику **Бурровс-Дельта** разом із п'ятьма індикаторами ризику, що дає інтегральну оцінку та грейд загрози.

Ця дизайн-система — бренд і UI-фундамент DIMS: токени, шрифти, повторно використовувані компоненти та високоточні відтворення двох поверхонь продукту.

> **Про мову.** DIMS — україномовний продукт (інтерфейс, копірайт і звіти українською). Увесь текст компонентів і вміст UI-кітів це відтворюють. Створюйте артефакти DIMS українською, якщо не попросять інакше.

## Продукт коротко

Аналітик додає **джерела** (завантажені файли, вставлений текст/URL, результати пошуку Google News / RSS / Telegram), налаштовує **параметри аналізу** (кількість MFW, поріг подібності θΔ, тип ознак, проєкцію) і запускає аналіз. На виході: **смуга статистики**, **підозрілі пари** (тексти, що стилістично надто подібні = можлива координація), інтегральна оцінка **R_DIMS** та **грейд загрози**. Окремий **редакційний звіт** подає повний розклад у форматі, готовому до друку.

Формула:

```
R_DIMS = w₁·I_content + w₂·I_coord + w₃·I_dynamics + w₄·I_impact + w₅·I_source
```

Два продукти / дві поверхні:
1. **Робоче середовище аналізу** — операційний інтерфейс із темною/світлою темами. Щільний, двоколонковий, відчуття приладової панелі.
2. **Аналітичний звіт** — редакційний лонгрід на теплому папері (FT / Economist / науковий журнал).

## ОСНОВИ КОНТЕНТУ

- **Мова:** українська скрізь. Російськомовний *вихідний матеріал* — це нормально (продукт моніторить РФ-медіа), але власний голос продукту — українська.
- **Регістр:** інституційний, точний, стриманий — військово-аналітичний, не маркетинговий. Без гучності, без окликів, без звертання на «ти/ви». Продукт констатує висновки, а не переконує.
- **Особа й голос:** безособово / у третій особі. Висновки повідомляються («Виявлено підвищений ризик», «Рекомендується пріоритетне опрацювання»), ніколи не «ви» / «ми радимо».
- **Регістр літер:** усе — реченнєвим регістром (заголовки, кнопки, мітки). ВЕЛИКІ ЛІТЕРИ — лише для дрібних трекінгових мета-міток (`ПІДОЗРІЛИХ ПАР`) та кікера звіту (`DIMS · АНАЛІТИЧНИЙ ЗВІТ`). Коди грейдів — великі токени (F/B/S/SS/SSS).
- **Числа й одиниці:** завжди присутні, завжди табличні. Дельти до 3–4 знаків (`Δ = 0.2410`), оцінки до 2–4 (`R_DIMS = 0.7400`), кількість токенів із розрядкою (`4 210 ток.`). Довірчі інтервали — моноширинним (`95% CI [0.198, 0.284]`).
- **Хеджування обов'язкове.** Стилометрія подається як *індикатор, а не доказ* — текст послідовно це вмотивовує («індикативні», «потребують верифікації», «не є доказом»). Це методологічна й етична позиція, вшита в голос.
- **Термінологія:** наукова й послідовна — *стилометрія, Δ-Бурровс, MFW, culling, проєкція, грейд, індикатор, координація*. Математична нотація (`Δ_Burrows`, `θΔ`, `I_content`) набирається курсивним серифним math-накресленням.
- **Емодзі:** жодних у тексті продукту. Єдиний знак, що вживається семантично, — **⚑** (прапор) для позначення ескалації SS/SSS. Декоративних емодзі немає ніколи.
- **Приклади мікрокопірайту:** `Додайте мінімум 2 джерела для запуску` · `URL → автозавантаження · HTML → очищується · Текст → напряму` · `Менше Δ = більша подібність. Рекомендовано: 0.6–1.0.`

## ВІЗУАЛЬНІ ОСНОВИ

**Дві поверхні, один бренд.** Вони навмисно різні й не повинні змішуватися: робоче середовище — холодний, темний, щільний прилад; звіт — теплий, світлий, просторий документ.

### Робоче середовище (операційний інтерфейс)
- **Колірний характер:** холодний шифер. Світла тема на `#f7f9fc`; темна тема (фірмовий вигляд) на шиферних `#0f172a → #29384f`. Чотири фонові шари, три ваги бордюрів, чотири ваги тексту — усі перемикаються через `[data-theme]`.
- **Акцент:** єдиний **сигнальний синій** для основних дій (`#1f6feb` світла / `#60a5fa` темна) та **бірюзовий** для посилань/допоміжного (`#0f766e` / `#2dd4bf`). Акцент уживається ощадливо — це не барвистий UI.
- **Сигнальні кольори:** зелений/бурштиновий/помаранчевий/червоний/фіолетовий для операційного статусу — лише на даних (бейджі, лінії підозрілих пар, числа статистики), ніколи як фон.
- **Типографіка:** **DM Sans** для всього UI (400–800), **JetBrains Mono** для всіх даних, метаданих, метрик та ID. Тіло — дрібне й щільне (13px). Табличні цифри скрізь, де є числа.
- **Фони:** лише плоскі заливки. **Без градієнтів** будь-де, окрім банера критичної ескалації SS/SSS (горизонтальний червоний градієнт — єдиний навмисний градієнт у системі). Без зображень, текстур, ілюстрацій.
- **Кути:** тісні. `4px` на контролах (кнопки, поля, бейджі), `8px` на контейнерах (картки, панелі, таби), `20px` пігулка лише на під-режимних перемикачах. Нічого м'якшого — це прилад, а не споживчий застосунок.
- **Картки:** заливка `--bg-1`, волосяний `1px` `--border-0`, радіус `8px`, `--shadow-sm` (ледь помітний `0 1px 2px`). Смуга статистики використовує **сітку з проміжком 1px над фоном кольору бордюра** як роздільники — без бордюрів на окремих комірках.
- **Тіні:** лише два рівні. `--shadow-sm` для карток у спокої; `--shadow-md` (м'який `0 12px 28px`) для піднятих/плаваючих елементів (ховер кнопки аналізу, шухляди, тости, критичний банер). Темна тема поглиблює обидва.
- **Бордюри й блюр:** липкий хедер використовує напівпрозорий `--bg-1` із `backdrop-filter: blur(14px)`. Оверлеї/спінери — темна заслінка + `blur(6px)`. Блюр функціональний (відокремлення хрому), ніколи декоративний.
- **Ховер/натискання:** ховер поглиблює заливку на один фоновий шар (`bg-2 → bg-3`) і темнить бордюри на крок; посилання отримують бірюзу + підкреслення. Основна кнопка темніє через `color-mix(…86%, #000)`. Кнопка аналізу — єдиний елемент, що *підіймається* на ховері (`translateY(-1px)` + тінь) і повертається при натисканні. Без масштаб-стрибків, без сяйва.
- **Анімація:** швидка й утилітарна — `120–150ms` на колір/фон/бордюр; `200ms` на ширину прогресу; `220ms` ease на шухляду журналу; `0.8s` лінійний спінер. Згладжування — типове/лінійне. Без пружин, без паралаксу. Відповідає стриманому тону.
- **Фокус:** контур `2px` `--accent` зі зсувом `2px` — видимий, доступний, ніколи не прибраний.
- **Розкладка:** фіксований липкий хедер `60px`; двоколонкове робоче середовище — липка ліва панель `360–420px` (вхід + корпус) над текучою правою панеллю (параметри + запуск + результати). Ліва панель скролиться незалежно.

### Звіт (редакційна поверхня)
- **Колірний характер:** теплий папір. Папір `#FDFDF8`, чорнило `#14110F`, градуйований теплий сірий неутральний ряд, волосяні лінії `#D9D2C2`.
- **Акценти:** приглушені журнальні тони — нейві `#0F2A4A`, кармін `#9B1B1B` (секційні кікери, цифри-краплі), бурштин, ліс, бірюза. Кармін — редакційний підпис.
- **Типографіка:** **Source Serif 4** для заголовків і тіла (голос лонгріду), **Inter** для міток/кікерів/мета, **JetBrains Mono** для дельт і матриць. Просторий міжрядковий `1.65`, міра `820px`.
- **Структура як декор:** працюють лінії — верхній бордюр `3px` + нижній `1px` на шапці, `3px double` навколо резюме-вставки, `1.5px` під заголовками таблиць, нумеровані рекомендації. Майже без заливок; сторінка зведена з чорнильних ліній на папері.
- **Кути:** практично прямі (`0–3px`). Це друк, а не UI.
- **Друк передусім:** звіт спроєктовано для чистого експорту в PDF (`@media print` згортає хром, робить грейд-бейджі контурними чорним по білому, уникає розривів усередині секцій).

### Спільне
- **Грейд-бейдж** — фірмовий елемент даних на обох поверхнях: моно (робоче середовище) / сан (звіт) код-чіп, забарвлений за рівнем загрози: F сірий · B блакитний · S бурштин · SS червоний · SSS темно-карміновий із внутрішнім кільцем. У друці рендериться контурним.
- **Математична нотація** (`Δ_Burrows`, `θΔ`, `R_DIMS`) завжди курсивна серифна math із під/надрядковими індексами — наскрізний мотив бренду.

## ІКОНОГРАФІЯ

DIMS **майже безіконний за задумом** — навмисна стриманість, що підсилює відчуття приладу/документа. Немає іконкового шрифту, немає набору SVG-іконок, немає бібліотеки іконок (Lucide/Heroicons тощо) у вихідниках.

- **Гліфи замість іконок.** Кілька вживаних символів — типографічні: **⚑** (прапор) для ескалації SS/SSS, **✕** для видалення/закриття, **↔** для «пари», **→/←** для навігації/посилань, **▸** для розкривної довідки (`details`). Каретка селекта — крихітний інлайн-SVG-шеврон, вшитий у `background-image` контрола.
- **Знак бренду** — типографічна монограма: квадратний бейдж «**D**» `32px` (заливка `--text-0`, гліф `--bg-1`, моно 800) поруч зі словесним знаком **DIMS** (DM Sans 800) із літерою **S** в акцентному кольорі. Інвертується між темами. Файла-логотипа немає — відтворюйте з розмітки (див. `guidelines/brand-logo.html`).
- **Статус несуть колір + типографіка, а не іконки** — грейд-бейджі, сигнально-забарвлені числа статистики, кольори лівих ліній сповіщень, забарвлені за каналом бейджі джерел (Google/RSS/Telegram).
- **Без емодзі.** Без растрових/PNG-іконок. Без декоративних ілюстрацій будь-де.

Якщо збірка справді потребує лінійних іконок (рідко), дотримуйтесь стриманості: тонкий одноваговий штриховий набір (напр. Lucide `1.5px`) у `--text-2`, лише там, де гліф не годиться, — і позначте додавання як відхід від вихідників.

## Шрифти

Усі чотири сімейства подаються з **Google Fonts** (локальні бінарники не потрібні): **DM Sans**, **JetBrains Mono** (робоче середовище) та **Inter**, **Source Serif 4** (звіт). Math-накреслення — **Cambria Math**, системний шрифт із серифним резервним стеком; якщо у вас є ліцензований math-вебшрифт, замініть `--font-math` у `tokens/typography.css`.
