# Протокол статистичної валідації DIMS (Фаза 2)

> Документ описує процедуру **емпіричної перевірки точності** удосконаленої
> методики DIMS на розміченому експертами наборі подій та **калібрування
> порогів грейдів за ROC/Youden** замість рівномірних. Це закриває два
> залишкові закиди експертної ради: «рівномірні пороги довільні» та «немає
> підтвердження точності». Інструмент реалізовано в `tools/validation.py`;
> усі числа відтворювані. **Жодних даних не вигадано** — звіт формується лише
> з наданого автором набору.

---

## 1. Що валідуємо

DIMS присвоює інформаційній події грейд $G \in \{F, B, S, SS, SSS\}$ на основі
інтегрального показника $R_{DIMS} \in [0,1]$ з порогами. Валідація відповідає
на питання: **наскільки автоматичний грейд DIMS збігається з експертним**, і
**які пороги $\theta_k$ оптимальні** за даними (а не призначені рівномірно).

## 2. Одиниця розмітки — подія

Розмічається **подія** (а не окреме джерело), бо $R_{DIMS}$ — корпусний показник.
Подія = набір джерел на одну тему + один експертний грейд. Формат — JSONL
(`Data/validation_manifest_template.jsonl`):

| Поле | Опис |
|---|---|
| `event_id` | унікальний ідентифікатор події |
| `grade` | **експертний** грейд F/B/S/SS/SSS (ground truth) |
| `manifestation` | опц.; вид прояву (`taboo` → авто-SS за Наказом № 46) |
| `sources[].text` / `text_path` | текст джерела або шлях до файлу |
| `sources[].domain` | домен (для $I_{source}$) |
| `sources[].published` | дата публікації ISO/RFC822 (для $S_{time}$) |

## 3. Процедура розмітки (рекомендації)

1. **Незалежність.** Бажано ≥ 2 експерти; розбіжності розв'язувати третім або
   консенсусом. Фіксувати міжекспертну згоду (та сама зважена κ).
2. **Сліпість.** Експерт виставляє грейд, **не бачачи** оцінки DIMS.
3. **Збалансованість.** Прагнути представленості всіх п'яти грейдів (особливо
   рідкісних SS/SSS), інакше метрики й ROC за відповідними межами нестійкі.
4. **Обсяг.** Орієнтовно **100–200 подій** для стійких оцінок; мінімум — кілька
   десятків. Менший набір дає лише ілюстративні, а не статистично значущі
   висновки (зазначати це чесно).
5. **Походження.** Реальні події моніторингу (зокрема матеріали КШН-2025) з
   доменами та позначками часу, щоб працювали $I_{source}$ і $S_{time}$.

## 4. Метрики

| Метрика | Що показує | Орієнтир |
|---|---|---|
| **Точність (exact)** | частка точних збігів грейду | контекстний |
| **Точність ±1 рівень** | збіг у межах сусіднього грейду | толерантна оцінка |
| **Зважена κ Коена (quadratic)** | узгодженість на **порядковій** шкалі з поправкою на випадковість | 0,41–0,60 помірна; 0,61–0,80 суттєва; > 0,80 майже повна |
| **MAE (індекс грейду)** | середнє відхилення в рівнях | менше = краще |
| **Матриця плутанини 5×5** | де саме методика помиляється | діагностика |

Квадратична κ обрана навмисно: помилка «F замість SSS» штрафується сильніше,
ніж «S замість SS» — це відповідає порядковій природі шкали інтересу.

## 5. Калібрування порогів за ROC/Youden

Для кожної межі грейдів («≥ грейд $k$») будується ROC-крива за $R_{DIMS}$, а
оптимальний поріг $\theta_k$ обирається за **індексом Юдена** $J = TPR - FPR$
(точка максимальної роздільної здатності). Додатково фіксується **AUC** —
інтегральна якість розділення за цією межею.

Результат — таблиця «чинний (рівномірний) θ vs ROC-калібрований θ»:
- якщо ROC-пороги близькі до рівномірних → рівномірні **емпірично виправдані**;
- якщо суттєво відрізняються → методику слід **переналаштувати** на ROC-пороги,
  що прямо знімає закид про довільність.

Висока AUC (напр. > 0,8) за межами — самостійний аргумент: $R_{DIMS}$ добре
розділяє рівні інтересу.

## 6. Запуск

```bash
# Реальна валідація (коли є розмічений набір):
.venv/bin/python tools/validation.py --manifest Data/validation_set.jsonl

# Перевірка працездатності коду на синтетиці (НЕ валідація):
.venv/bin/python tools/validation.py --selftest
```

Виходи: таблиці (Markdown) у stdout + матриця плутанини
`docs/img/validation_confusion.png`.

## 7. Інтерпретація для дисертації (Розділ 3)

Результати валідації подаються в експериментальному розділі:
1. Таблиця метрик (точність, ±1, κ, MAE) — кількісне підтвердження якості.
2. Матриця плутанини — діагностика типових помилок.
3. Таблиця ROC-порогів + AUC — обґрунтування (або переналаштування) шкали.
4. Якщо ≥ 2 експерти — міжекспертна κ як верхня межа досяжної згоди.

## 8. Обмеження

- Висновки достовірні лише в межах представленості грейдів у наборі.
- Для рідкісних SS/SSS потрібна цілеспрямована вибірка, інакше ROC за верхніми
  межами нестійкий.
- Валідація оцінює узгодженість DIMS з **експертом**, а не «абсолютну істину»;
  тому ключова й міжекспертна згода як орієнтир стелі.
