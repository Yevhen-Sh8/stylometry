Прочитайте файл `readme.md` у цьому скілі та дослідіть інші доступні файли.

Якщо створюєте візуальні артефакти (слайди, моки, одноразові прототипи тощо) — скопіюйте ассети назовні та зробіть статичні HTML-файли для перегляду користувачем. Якщо працюєте з продакшн-кодом, можете копіювати ассети й читати правила тут, щоб стати експертом із дизайну в цьому бренді.

Якщо користувач викликає цей скіл без додаткових вказівок, запитайте, що він хоче збудувати чи спроєктувати, поставте кілька запитань і дійте як експерт-дизайнер, що видає HTML-артефакти _або_ продакшн-код — залежно від потреби.

> **Мова.** DIMS — україномовний продукт. Увесь текст продукту створюйте українською, якщо не попросять інакше.

## Швидке орієнтування

- **`readme.md`** — повний дизайн-гайд: вихідні матеріали, основи контенту, візуальні основи (дві поверхні — холодне темне **робоче середовище** та тепле світле **звіт**), іконографія та покажчик усіх токенів, компонентів і UI-кітів.
- **`styles.css`** + **`tokens/`** — залінкуйте `styles.css`, щоб успадкувати всі CSS-кастом-властивості та вебшрифти. Токени охоплюють обидві поверхні й три теми (`[data-theme="auto|light|dark"]`).
- **`components/`** — React-примітиви (`Button`, `GradeBadge`, `Badge`, `Input`, `Select`, `Tabs`, `StatCard`, `SourceItem`, `Alert`). Кожен має `.prompt.md` з прикладом використання. Монтуйте з зібраного бандла через `window.DIMSDesignSystem_175874`.
- **`ui_kits/workspace/`** та **`ui_kits/report/`** — повноекранні відтворення для копіювання.
- **`guidelines/`** — картки-зразки основ.

## Непорушні правила

- **Текст українською**, інституційний/аналітичний регістр, без звертання на «ти/ви», без гучності, без емодзі (лише **⚑** для ескалації SS/SSS). Завжди хеджуйте стилометричні твердження («індикатор, не доказ»).
- **Не змішуйте дві поверхні.** Робоче середовище = DM Sans + JetBrains Mono, холодний шифер, щільно, тісні радіуси. Звіт = Source Serif 4 + Inter, теплий папір, просторо, лінії.
- **Грейд-бейдж (F→B→S→SS→SSS) — фірмовий елемент.** Табличні цифри скрізь, де є числа. Плоскі заливки (єдиний градієнт — критичний банер). Тісні кути (4px контроли / 8px контейнери).
