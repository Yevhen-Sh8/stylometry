# Розмічений набір для валідації DIMS (Фаза 2)

Тут збирається **розмічений експертами набір подій** для статистичної валідації
методики (`docs/validation_protocol_ua.md`, інструмент `tools/validation.py`).

## Структура

```
Data/validation_set/
├── README.md                     ← цей файл
├── manifest.jsonl                ← головний реєстр подій (це заповнюєш)
└── events/
    ├── ev001_EXAMPLE/            ← приклад події (видали або заміни)
    │   ├── src_01_ria.txt
    │   └── src_02_tass.txt
    ├── ev002/
    │   └── ...
    └── ...
```

## Як додати подію

1. Створи теку `events/<event_id>/` (напр. `events/ev017/`).
2. Поклади туди **тексти джерел** окремими `.txt` (мінімум **2** на подію — цього
   вимагає Burrows' Delta). Назва файлу довільна; зручно `src_NN_<домен>.txt`.
3. Додай **один рядок** у `manifest.jsonl`:

```json
{"event_id": "ev017", "grade": "S", "manifestation": "fake", "sources": [
  {"text_path": "Data/validation_set/events/ev017/src_01_ria.txt", "domain": "ria.ru", "published": "2026-05-01T10:00:00"},
  {"text_path": "Data/validation_set/events/ev017/src_02_tass.txt", "domain": "tass.ru", "published": "2026-05-01T10:35:00"}
]}
```

- `grade` — **експертний** грейд (ground truth), один із: `F`, `B`, `S`, `SS`, `SSS`.
- `manifestation` — опційно; вид прояву. `taboo` → авто-SS (за Наказом № 46).
- `domain` — для індикатора ризику джерела `I_source` (бажано).
- `published` — дата публікації (ISO `2026-05-01T10:00:00` або RFC822 з RSS) — для `S_time`.

## Шкала грейдів (Наказ МО України № 46)

| Грейд | Рівень інтересу |
|---|---|
| **F** | Малий — зберігання без опрацювання |
| **B** | Точковий — обмежене опрацювання |
| **S** | Однозначний — опрацювання + пропозиції щодо реагування |
| **SS** | Вагомий — повідомлення керівника, якнайшвидше опрацювання |
| **SSS** | Критичний — позачергове опрацювання, негайне реагування |

## Види прояву (поле `manifestation`, опційно)

`manipulation`, `fake`, `sandwich`, `conspiracy`, `insider`, `sensitive`, `taboo`.
Вид прояву **на бали не впливає** (це категоріальна ознака); виняток — `taboo`
(авто-грейд не нижче SS).

## Рекомендації щодо розмітки

- **≥ 2 експерти**, сліпо (не бачачи оцінки DIMS); розбіжності — консенсусом.
- **Збалансованість** грейдів: цілеспрямовано додавай рідкісні `SS`/`SSS`.
- Обсяг: **~100–200 подій** для стійких метрик і ROC-порогів.
- Реальні матеріали моніторингу (зокрема КШН-2025) — пріоритет.

## Перевірка та запуск

```bash
# 1) Перевірити коректність manifest (грейди, ≥2 джерела, наявність файлів):
.venv/bin/python tools/validation.py --check --manifest Data/validation_set/manifest.jsonl

# 2) Запустити валідацію (метрики + ROC-пороги + матриця плутанини):
.venv/bin/python tools/validation.py --manifest Data/validation_set/manifest.jsonl
```

> Приклад `ev001_EXAMPLE` — заглушка для демонстрації структури. Перед реальним
> прогоном видали його з тек і з `manifest.jsonl`.
